# 엣지 케이스 / 예외 테스트

비정상적이거나 모호한 입력에 대한 Agent의 처리를 테스트한다.

## Goal #26 — 모호한 groupBy 표현

- **Goal**: "카테고리별 통계를 보여줘"
- **기대 동작**: LLM이 "카테고리"를 provider/source_type/update_type 중 하나로 해석하거나 사용자에게 확인 질문
- **검증 포인트**: 모호한 표현에 대해 합리적인 판단 또는 clarification 요청

## Goal #27 — 매우 짧은 기간 (1일)

- **Goal**: "어제 데이터 통계 알려줘"
- **기대 파라미터**: `startDate="(어제)"`, `endDate="(어제)"`, groupBy는 LLM 판단
- **검증 포인트**: 1일 범위 데이터 정상 조회, 데이터 없을 경우 빈 결과 처리

## Goal #28 — 데이터 없는 과거 기간

- **Goal**: "2020년 데이터 통계를 보여줘"
- **기대 파라미터**: `startDate="2020-01-01"`, `endDate="2020-12-31"`, groupBy는 LLM 판단
- **검증 포인트**: totalCount=0, groups=[] 등 빈 결과를 사용자에게 자연스럽게 안내

## Goal #29 — 미래 기간 조회

- **Goal**: "2027년 Provider별 통계를 보여줘"
- **기대 파라미터**: `groupBy=provider`, `startDate="2027-01-01"`, `endDate="2027-12-31"`
- **검증 포인트**: 미래 날짜 입력 시 오류 없이 빈 결과 반환

## Goal #30 — groupBy 미지정 (전체 분석 요청)

- **Goal**: "전체 데이터를 분석해줘"
- **기대 동작**: LLM이 스스로 적절한 groupBy를 선택하거나 여러 groupBy로 다중 호출
- **검증 포인트**: 명확한 groupBy 없이도 유의미한 분석 결과를 제공하는지 확인
